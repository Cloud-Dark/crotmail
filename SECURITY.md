# Security Policy

## Supported Versions

Saat ini branch `main` dianggap sebagai versi yang didukung untuk perbaikan keamanan.

## Reporting a Vulnerability

Jika menemukan kerentanan keamanan, jangan laporkan detail eksploit secara publik di issue biasa.

Gunakan salah satu jalur berikut:

1. GitHub Security Advisory (preferred)
2. Private contact ke maintainer repository

Saat melapor, sertakan informasi berikut:

- Ringkasan kerentanan
- Langkah reproduksi
- Dampak potensial
- Saran mitigasi (jika ada)

## Response Expectations

Target respons awal:

- Konfirmasi penerimaan laporan: maksimal 72 jam
- Update status investigasi: maksimal 7 hari
- Rilis perbaikan: secepat mungkin sesuai tingkat risiko

## Disclosure Policy

- Laporan akan ditinjau dan diverifikasi terlebih dahulu.
- Jika valid, patch akan disiapkan sebelum pengungkapan publik.
- Kredit pelapor dapat dicantumkan (opsional, sesuai persetujuan pelapor).
